## Spec compliance
(a) PinEntry gains ContentID/State/LastError; Ready atomic.Bool kept as lock-free
    fast path, always equal to (State=="ready"). IsReady semantics unchanged
    (Ready fast path AND storage.Has).
(b) Codec upgraded: pinEntryJSON carries content_id/state/last_error (omitempty)
    plus the legacy ready bool. decodePinEntry maps legacy records (state==""):
    ready=true→"ready", ready=false→"pulling". Existing BadgerDB data loads.
(c) ApplyPin(blobHash, blobType, role string, size int64, contentID string):
    initial State=pulling; fetchPinnedBlob success→ready; failure→failed +
    LastError=err.Error() truncated to 512 chars (rune-safe). Panic path also
    → failed ("fetch panicked: ...").
(d) List(filter PinFilter) []PinEntry and Get(blobHash) (PinEntry, bool) added.
    PinFilter: Ready *bool (nil=any; false matches pulling+failed), Role,
    ContentID. Snapshots taken under stateMu.
(e) RetryPin(blobHash) bool: CAS failed→pulling under stateMu (LastError
    cleared), then `go fetchPinnedBlob`. Idempotent: second call while pulling
    returns false; ready/missing → false.
(f) Call sites adapted: handler.go (metas → update.ContentID, legacy → ""),
    store_test.go, handler_test.go, cluster_test.go.

Must-NOT honored: BadgerDB key structure unchanged ("p:"+hash); ApplyUnpin
semantics untouched; three states only, no progress percent.

## Concurrency design
State/LastError are plain strings (spec-fixed) mutated by fetch goroutines and
read by List/Get → guarded by new PinStore.stateMu (sync.RWMutex).
setPinState(blobHash, from, to, lastErr) does the compare-and-set + persist
atomically under stateMu; Ready.Store is always updated in the same critical
section, keeping Ready == (State=="ready") invariant. encodePinEntry is only
called under stateMu or before the entry is shared.

## Deviations
1. test/integration/cluster_test.go:883 updated to the 5-arg ApplyPin. Spec
   (f) says "Adapt ALL call sites (… tests, etc.)"; CI runs `go vet ./...`
   (workflows/ci.yml:32) which compiles test files, so leaving the 4-arg call
   would break CI for every agent. One-argument mechanical update only.
2. go vet copylocks: List/Get return PinEntry BY VALUE per the spec-mandated
   signatures while PinEntry embeds atomic.Bool. Copy copies are of fresh,
   never-shared values (false-positive class); code uses named result + naked
   return in snapshotPinEntry so `go vet ./...` stays clean.
3. ApplyPin now has 5 params (>3-param guideline) — spec-mandated exact
   signature, documented in the function docstring.
