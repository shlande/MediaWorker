## Design decisions (locked)
- Instrumentation is explicit in-handler calls (NOT middleware) so result is
  known after the op. Rule: audit at every terminal AFTER the body parses as
  JSON; result=ok on 2xx, fail otherwise; unparseable JSON -> no audit.
- Read operations are NOT instrumented (GET accounts/whitelist/contents/
  nodes/me have zero Record calls).
- recordWriteAudit has 7 params by design: it is THE instrumentation funnel,
  params map 1:1 onto AuditEntry fields; an options struct would be the
  "throwaway config object" workaround the >3-param smell rule itself counts.
  30 call sites stay one-liners.
- RegisterAuthRoutes NOT extended: it already reads srv.audit at registration
  (todo 18); todo 54 must call SetAuditRecorder BEFORE RegisterAuthRoutes.
  All other Register*Routes take audit as the LAST param (nil = disabled).
- main.go untouched (D1); recorder construction + SetAuditRecorder + route
  mounts consolidate in todo 54. Todo 34 reads via mc.ListAdminAudit.
