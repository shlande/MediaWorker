### Audit ordering (todo 33's note — VERIFIED, load-bearing)
- READ server.go + auth_handlers.go: RegisterAuthRoutes captures `srv.audit` at
  REGISTRATION time (auth_handlers.go:169,171 — `loginHandler(users, srv.secret, srv.audit)`,
  `logoutHandler(srv.audit)`). It is NOT read at request time.
- Therefore `adminSrv.SetAuditRecorder(...)` MUST precede RegisterAuthRoutes.
  Implemented: auditInserter/auditRec built first inside the admin block, then
  SetAuditRecorder, then ALL route mounts (auth routes mount later inside `if mc != nil`).

  (NewPGAuditRecorder(nil) is a documented no-op recorder; a typed-nil
  *PGMetadataClient inside the interface would panic on Record).

decorator: `quotaAwareAccountsWriter` (package main) embeds
adminapi.AdminAccountsWriter and overrides CreateAccount (non-zero rate_limit)
+ SetRateLimit to call `qa.SetGlobalLimit("vendor:account_id", cfg)` AFTER the
registry write succeeds. Handler logic untouched (MUST-NOT honored).
